# Playbooks de Agentes

Esta pasta mantém apenas os playbooks úteis para o fluxo real do projeto.

## Agentes mantidos

- [feature-developer.md](./feature-developer.md): implementação ponta a ponta respeitando boundaries do repo
- [code-reviewer.md](./code-reviewer.md): revisão por risco, regressão e aderência às regras
- [frontend-specialist.md](./frontend-specialist.md): execução e revisão de UI/UX
- [security-auditor.md](./security-auditor.md): auth, tenant scope, dados sensíveis e integrações externas
- [supabase-expert.md](./supabase-expert.md): schema, migrations, RPC, RLS e sincronização de tipos Supabase

## Uso esperado

- Leia `AGENTS.md` primeiro.
- Use `.context/docs/README.md` para descobrir o doc complementar da tarefa.
- Use estes playbooks só quando a tarefa realmente pedir esse papel; eles não substituem as regras canônicas.
